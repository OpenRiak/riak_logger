Description

Riak kernel logger configuration and operations.

Filter functions are defined as 
   filter_<codes>(Event :: logger:log_event(), Action :: stop | log)
       -> logger:log_event() | stop | ignore.
where <codes> is an alphabetically-ordered list of 
   c - match proc_lib-style crash reports
   e - match emulator-spawned process crashes
   p - match application/supervisor progress reports
   r - match events from processes whose group leader is remote
   s - match SASL reports

This allows a single filter to be specified in a logger handler specification, reducing or eliminating filter chaining to improve performance.

Note that some non-composite filters are semantically equivalent to functions in the OTP logger_filters module and are documented as such. In all cases the filters defined here should outperform their OTP counterparts due to explicit function head pattern matching.
